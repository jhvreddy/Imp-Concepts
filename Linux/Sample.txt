How can we display content of all the .txt files with one command in a directory

find . -type f -name "app.txt" | xargs cat ( "xargs" takes input from strandard input (stdin) and turns it into arguments for another command )

find . -type f -name "app.txt" | xargs cat | grep "ERROR"

find . -type f -name "app.txt" | xargs cat | grep "ERROR" | sort (time ascending)

find . -type f -name "app.txt" | xargs cat | grep "ERROR" | sort -k4 (instead of time stamp, sort based on our required field, here it is 4)

find . -type f -name "app.txt" | xargs cat | grep "ERROR" | sort -k4 | uniq (It checks only entire line is unique/not)

find . -type f -name "app.txt" | xargs cat | grep "ERROR" | sort -k4 | uniq -f3 (check the uniqness from specific field)


find . -type f -name "app.txt" | xargs cat | grep "ERROR" | sort -k4 | uniq -f3  > errors.log


find . -type f -name "*.txt" -exec cp {} /Users/nana/backup \;
exec tells linux to execute a specific command for each file that it finds

find . -type f -name "*.txt" -exec cp rsync -R {} /Users/nana/backup \;
rsync used for efficiently copying and synchronizing files and directories between a source and destination (widely used during backup)
